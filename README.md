<p>編集後、mainのブランチへ反映させるだけで自動的にデプロイされます。<br /></p>

## 命名規則
<p>単語をハイフン（-）で区切る形式（ケバブケース）を採用します。　例：kebab-case<br />
命名規則とは、クラスや変数などの名前を自分で決める時のルールです。みんなに伝わるわかりやすい名付けを心がけましょう。<br /></p>

## コミットメッセージの書き方
<p>コミットメッセージは以下の様に書いてください<br />
[add]:機能追加<br />
[update]:機能更新<br />
[fixed]:機能修正<br />
[finish]:機能完成<br />
[delete]:機能削除<br />
</p>
